下篇：零成本防封自主抓取方案 (Geek Track)

商业 API 虽好，但往往伴随高昂费用和 QPS 限制。为了实现“免费多源交叉验证”的架构，我们逆向分析了全球几大顶级媒体和比分软件的无鉴权（或轻量鉴权）API 接口，形成了一套**不依赖 Selenium 等低效爬虫、直接窃听 JSON Payload 的合法极客抓取方案**。

ESPN Hidden API 极力推荐

ESPN 为其自身 App 和官网构建的隐藏 RESTful API，是目前互联网上最稳定、最庞大且无需任何 API Key 即可无限次 GET 的宝藏接口。

  • 延迟：约 10-15 秒（进球后）。
  • 包含：比分、比赛时钟、赛程。
GET https://site.api.espn.com/apis/site/v2/sports/soccer/fifa.world/scoreboard

FotMob App JSON API 数据极度丰富

FotMob 作为全球第一大比分 App，其底层 Web/App 共用了一套极为清晰的 API。它包含球员头像、xG 进球期望甚至传球网络图。

  • 反爬：极弱，只需伪造常规 User-Agent 即可。
  • 包含：包含极其深入的比赛数据流。
GET https://www.fotmob.com/api/matchDetails?matchId=xxxxxx

BBC Sport GraphQL / JSON 英国最稳

BBC 体育的数据频道拥有极高的权威性。其底层使用一套清晰的推送流。

  • 延迟：文字直播和比分大约 30 秒左右延迟。
  • 包含：极其权威和详尽的英文文字解说流。
GET https://push.api.bbci.co.uk/batch?t=%2Fdata%2Fbbc-morph-sport-live-scores...

SofaScore / Flashscore 对抗激烈

这类比分提供商的数据极为实时，但其反爬机制相对较强。

  • 反爬：存在动态 Token 验证、Cloudflare 防护、IP 封锁策略。
  • 建议：不建议作为主程序的轮询抓取源，维护成本大于购买商业 API。

💡 2026 世界杯落地建议 (Recommended Setup)

综合成本、可用性、数据覆盖与实施复杂度，推荐采用“1 个付费主源 + 2 个免费验证源 + 统一裁决引擎”的组合方案：

  1. 主力买单（付费兜底）： 购买 $19/月的 API-Football 作为主 Source A。它成本低、接入快，可提供稳定的基础数据保障。
  2. 自主抓取（免费高速验证）： 将 ESPN Hidden API 和 FotMob API 封装为 Source B 和 Source C，用于免费补充与交叉验证。
  3. 引擎裁决： 由 Node.js Worker 每隔 15 秒并发请求这三个 JSON 接口，通过统一规则完成去重、冲突检测与结果裁决。由于全部采用 JSON 数据流而非 HTML 解析，CPU 开销低，容错性更高。
